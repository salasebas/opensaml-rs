# Migrating from 0.3 to 0.4

This guide covers the changes consumers need to make when upgrading from
`0.3` to `0.4`. For the complete list of changes, see the
[changelog](../../CHANGELOG.md).

## Contents

- [AuthnRequest `IssueInstant` is required](#authnrequest-issueinstant-is-required)

## AuthnRequest `IssueInstant` is required

Inbound `AuthnRequest` messages must now include an unqualified `IssueInstant`
with a valid UTC `xs:dateTime` value ending in `Z`. The check applies to HTTP
POST, HTTP-Redirect, and HTTP-POST-SimpleSign.

Custom request XML must include the attribute:

```xml
IssueInstant="2026-07-16T23:13:10Z"
```

Code that manually constructs `FlowResult` fixtures must also provide
`request.issueInstant`. Typed handlers can read the whitespace-normalized value
through `AuthnRequest::issue_instant()`, which returns the existing
`SamlInstant` type.

Missing, malformed, or non-UTC values now return
`SamlError::ProtocolProfile`. Valid older timestamps remain accepted; this
change validates presence and format without adding a freshness policy.
